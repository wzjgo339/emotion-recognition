# 表情识别系统项目总结报告

## 📋 项目概述

本项目是一个基于深度学习的七类表情识别系统，能够识别以下表情：`angry`（愤怒）、`disgust`（厌恶）、`fear`（恐惧）、`happy`（快乐）、`sad`（悲伤）、`surprise`（惊讶）、`neutral`（中性）。

### 🎯 核心功能
- 训练深度学习模型进行表情分类
- 单张图片实时表情识别
- 批量图片处理
- 模型性能评估与可视化
- 用户友好的交互界面

---

## 🧠 算法架构

### 1. 深度学习模型架构

#### 主要模型：CNNWithAttention
**输入：** `48×48` 灰度图像  
**输出：** 7类表情的概率分布

#### 网络结构：

**卷积层组（4层）：**
- **Conv1：** `1→64` 通道，3×3卷积核，BatchNorm + ReLU
- **Conv2：** `64→128` 通道，3×3卷积核，BatchNorm + ReLU  
- **Conv3：** `128→256` 通道，3×3卷积核，BatchNorm + ReLU
- **Conv4：** `256→512` 通道，3×3卷积核，BatchNorm + ReLU

**注意力机制：**
- **SelfAttention（自注意力）：** 捕获特征之间的长距离依赖关系
- **SEBlock（Squeeze-and-Excitation）：** 通道注意力机制，增强重要特征

**分类器：**
- 全局平均池化 + 三层全连接网络
- Dropout正则化防止过拟合
- 输出层：`512→256→128→7`

#### 模型参数量：约 4.2M

### 2. 训练策略

**优化器：** AdamW (`lr=0.001, weight_decay=1e-4`)  
**学习率调度：** ReduceLROnPlateau（动态调整）  
**损失函数：** CrossEntropyLoss（交叉熵损失）  
**正则化：** Dropout（0.5, 0.3, 0.2）+ L2正则化

**数据增强：**
- 随机旋转（±15°）
- 水平翻转
- 随机裁剪
- 亮度调整

### 3. 图像预处理

**标准化参数：**
- 均值：`[0.5076]`
- 标准差：`[0.2128]`

**预处理流程：**
1. 灰度转换
2. 人脸检测与对齐
3. 尺寸调整至 48×48
4. 归一化处理

## 🎯 性能表现

### 模型对比评估

我们对项目中的两个模型进行了详细对比评估：

| 指标 | best_model.pth | best_fixed_model.pth | 差异 |
|------|----------------|---------------------|------|
| **总体准确率** | **68.00%** | 66.17% | +1.83% |
| angry（愤怒） | 58.04% | 58.56% | -0.52% |
| disgust（厌恶） | 54.05% | **83.78%** | -29.73% |
| fear（恐惧） | 34.77% | **44.43%** | -9.66% |
| **happy（快乐）** | **87.15%** | 82.36% | +4.79% |
| sad（悲伤） | **58.46%** | 50.12% | +8.34% |
| surprise（惊讶） | 83.51% | **84.36%** | -0.85% |
| neutral（中性） | **76.24%** | 69.26% | +6.98% |

### 结论

**`best_model.pth` 整体表现更优**，推荐作为默认模型使用：
- 总体准确率更高 (68% vs 66.17%)
- happy、sad、neutral 识别率显著更高
- 在主要表情类别上更加均衡

**`best_fixed_model.pth` 的特点：**
- disgust 识别率极高 (+29.73%)，但该类别样本仅111张，参考价值有限
- fear 识别率较高 (+9.66%)

### 模型来源说明

#### best_model.pth
通过标准训练流程 `train.py` 生成，训练约50个epoch得到的最佳模型。

#### best_fixed_model.pth
通过 `quick_fix.py` 优化得到，主要优化策略包括：

1. **数据平衡处理**：分析训练数据分布，对样本较少的类别进行过采样处理
2. **学习率优化**：使用更合适的初始学习率和调度策略
3. **早停机制**：当验证准确率持续下降时自动停止训练，防止过拟合
4. **最佳检查点保存**：在验证准确率提升时保存模型权重
